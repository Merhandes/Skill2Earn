### 1. Pemilihan Teknologi Pengembangan Awal (21 November 2025)
Pengembangan versi pertama aplikasi dilakukan menggunakan **HTML, CSS, dan JavaScript murni**.  
Keputusan ini diambil karena:
- Waktu pengembangan yang terbatas.
- Fokus pada pembuatan prototype yang cepat dan mudah diuji.
- Fitur awal belum membutuhkan backend atau sistem kompleks.

---

### 2. Rencana Pengembangan Backend di Masa Depan
Seiring meningkatnya kebutuhan seperti manajemen user, data UMKM, dan pelatihan, backend akan menjadi bagian penting dari platform.  
Rencana teknologi backend:
- Menggunakan **PHP** atau framework seperti **Laravel**.
- Menambahkan autentikasi, manajemen database, dan REST API.
- Membuat dashboard dinamis untuk UMKM dan peserta pelatihan.

Tujuan pengembangan backend:
- Meningkatkan skalabilitas.
- Mengoptimalkan keamanan data.
- Mempermudah ekspansi fitur aplikasi.

---

### 3. Penggunaan Bootstrap untuk Efisiensi Frontend
Bootstrap digunakan untuk mempercepat proses styling dan membuat antarmuka lebih konsisten.  
Alasan pemilihan:
- Memudahkan pembuatan layout responsif tanpa banyak CSS manual.
- Tersedia komponen siap pakai seperti button, form, card, navbar, dll.
- Mengurangi kompleksitas styling vanilla CSS.
